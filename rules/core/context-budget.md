# Context Anxiety Rule (v1.0)

> [!IMPORTANT]
> 에이전트는 무한 루프에 빠지거나 똑같은 실패를 반복하여 컨텍스트 윈도우를 낭비하는 현상(Zombie Lock)을 스스로 감지하고 중단해야 합니다.

## 1. 턴 예산 (Turn Budget)
모든 작업에는 보이지 않는 '턴 예산'이 존재합니다.
- **표준 예산**: 한 작업 당 최대 15턴.
- **임계점**: 예산의 80%(12턴)를 소모했을 때 진행률이 50% 미만이라면 **위험 상태**로 간주합니다.

## 2. 자가 진단 및 리셋 프로세스
에이전트는 위험 상태가 감지되면 즉시 다음 행동을 수행합니다.

1. **고집 부리기 중단**: "한 번만 더 하면 될 것 같다"는 본능을 억제하고 즉시 코드 수정을 멈춥니다.
2. **상태 요약 및 압축**: 지금까지 파악한 원인, 시도했던 방법, 실패한 이유를 `pc_memory_write`를 통해 핵심만 요약 저장합니다.
3. **세션 리셋 (Handoff)**: 현재까지의 성과를 `pc_save_observation`에 남기고, 다음 에이전트(또는 다음 세션의 자신)가 깨끗한 컨텍스트에서 시작할 수 있도록 락을 해제하거나 도움을 요청합니다.

## 3. 방지 전략
- 3회 연속 동일한 에러 발생 시, 로직 수정 대신 `pc_capsule`로 주변 컨텍스트를 다시 읽거나 `diagnostics.md` 프로토콜을 활성화하십시오.
- 불필요한 파일 읽기나 중복된 `ls` 명령어를 줄여 토큰 소비를 최소화하십시오.
