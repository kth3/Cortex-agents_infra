# 기계적 억제 장치 및 워크플로우 강제 (Guardrails)

에이전트의 사전 학습 편향(Pre-trained Bias)과 무분별한 툴 호출을 막기 위한 시스템 제어 장치입니다.

## 1. 프롬프트 및 도구 통제 (Prompt Override)
- **내장 탐색 도구 제한**: `grep`, `find_by_name`, `bash` 등을 통한 무지성 텍스트 검색을 1차적으로 차단합니다. 
- **Cortex MCP 강제**: 프로젝트 도메인 구조 파악 시 반드시 `mcp_cortex` 파이프라인(`pc_capsule`, `pc_memory_search_knowledge` 등) 도구를 우대하여 사용해야 합니다.

## 2. 작업 절차 및 실행 안전성
- **위험 명령어 확인 (SafeToAutoRun 지양)**: 백그라운드 프로세스, 무거운 빌드, 데이터 삭제(`rm -rf` 등)는 사용자의 승인을 거쳐야 합니다. 자율 실행(AutoRun)으로 파괴적인 작업을 수행하지 마십시오.
- **Silent Shortcuts 처리 지침**: `settings.yaml` 등에 정의된 특정 사용자 트리거에는 불필요한 사족이나 긴 보고서 출력을 생략하고 즉시 턴을 종료합니다.
- **절대 경로 회피**: 프로젝트 이식성을 위해 가급적 워크스페이스 내의 상대 경로를 준수합니다.

> [!WARNING]
> 기계적 툴 통제 규칙은 단순한 권장 사항이 아니며, 에이전트의 컨텍스트 비대화를 막기 위한 핵심적인 제약입니다.
