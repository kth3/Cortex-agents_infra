# 지식 탐색 강제화 지침 (Mandatory Discovery)

에이전트는 코드 분석이나 비즈니스 로직 작성 시, 자신의 일반적인 LLM 지식으로 무작정 코드를 짜는 것을 엄격히 금지하며, 반드시 프로젝트에 종속된(Context-Bound) 자산을 우선적으로 확인해야 합니다.

## 1. No-Skill 현상 방지
- 새로운 도메인 영역에 진입하거나 기존 코드를 리팩토링하기 전, 반드시 다음의 MCP 엔진 툴 중 하나 이상을 사용하여 선행 조사를 수행해야 합니다:
  1. `pc_capsule`: 현재 모듈과 연관된 아키텍처/컨텍스트 파악.
  2. `pc_memory_search_knowledge`: 유사한 문제에 대해 기존에 약속된 스킬(Skill)이나 패턴 검색.

## 2. 프로젝트 컨벤션 우위
- 탐색 결과로 나온 프로젝트 내부의 컨벤션이나 예외 처리 표준, 규칙 등이 LLM이 알고 있는 "일반적인 글로벌 클린 코드 룰"과 충돌할 경우, 무조건 **프로젝트의 내부 규칙**이 승리(Priority)합니다.
- 단순 타이포 수정과 같은 긴급 메인터넌스가 아닌 이상, 사전 탐색을 생략하여 프로젝트 고유 패턴을 이탈한 결과물은 치명적인 인프라/논리 결함으로 취급됩니다.

> [!CAUTION]
> 분석 결과에 프로젝트 고유 컨벤션이 전혀 묻어나지 않은 범용 코드가 제안된다면, 이는 지식 탐색 강제 조항 위반입니다.
