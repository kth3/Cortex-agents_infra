# 자율 탐색 및 인지 우선순위 (Cognitive Priorities)

상위 지침(`GEMINI.md`)의 토큰 경제성 원칙을 바탕으로, 에이전트가 지식을 스스로 탐색해야 하는 구체적인 상황과 우선순위를 정의합니다.

## 1. 자율 탐색 트리거 (Discovery Triggers)
에이전트는 다음 상황 발생 시 즉시 `pc_memory_search_knowledge`를 수행하여 인지를 보정해야 합니다.
- **인프라 결함**: MCP 도구 호출 시 에러가 발생하거나 응답이 예상과 다를 때.
- **맥락 불일치**: 현재 작업 데이터(코드 등)가 과거 작업 이력이나 프로젝트 컨벤션과 충돌한다고 느껴질 때.
- **모호한 명령**: 사용자의 지시어가 프로젝트 내부 전문 용어나 특정 스킬을 암시할 때.

## 2. 인지 우선순위 (Cognitive Stack)
에이전트는 정보를 결합할 때 다음 순서를 엄수합니다.
1. **실시간 데이터**: 현재 세션의 대화 맥락, 작업 파일의 최신 코드.
2. **지능형 검색 결과**: `pc_` 도구를 통해 소환된 스킬 및 규칙 조각.
3. **영구 기억**: `patterns.md`나 DB에 저장된 과거 결정 사항.

---

> [!TIP]
> 진정한 지능은 모든 것을 기억하는 것이 아니라, `GEMINI.md`의 통제 하에 가장 적절한 도구를 호출하여 정확한 답을 '연결'하는 것입니다.
